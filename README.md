# Cosmos

Cosmos is an AI powered learning game for young children. There is no fixed
question bank. Instead, a child picks a biome, chooses an animal guide, and then
answers questions that an AI writes live, tuned to the child's age and how they
have been doing. The animal reads every question out loud with AI generated
speech, the AI grades open ended answers written in the child's own words, and a
chatty AI robot shopkeeper answers whatever the child asks. The whole experience
is built to feel playful, friendly, and readable for kids who may not be able to
read well yet, which is why almost everything is spoken aloud.

## AI at the core

AI is not a side feature here. It drives the actual gameplay. Four AI
capabilities work together:

1. **AI question generation.** Every question is written on demand by a language
   model for the exact topic, animal character, and age group in play. Nothing
   is pre written or pulled from a static list. The model is told not to repeat
   earlier questions and to adapt to the child's recent performance, so the
   session shapes itself around each child.
2. **AI answer evaluation.** Open ended text answers are graded by the model,
   not by string matching. It judges whether the child actually understood the
   idea, stays warm and lenient for younger kids, and writes friendly spoken
   feedback.
3. **AI text to speech.** Every question and every piece of feedback is turned
   into a natural voice through an AI speech service, so the game can be played
   by children who cannot read yet.
4. **AI robot shopkeeper.** The Space Shop has a robot that holds a real
   conversation and answers free form questions from the child through the same
   language model.

The most interesting part is how evaluation and generation combine. When a child
answers an open ended question, a single AI request both grades that answer and
writes the next question at the same time. The moment the spoken feedback
finishes, the next question is already generated and its voice audio is already
prefetched, so it appears and reads aloud with no waiting. This lives in
`evaluateTextAndGenerateNext` in `src/lib/ai.js`, and the sections below go into
the details.

## What the app does

- **Sign up and sign in.** Accounts are handled through Supabase auth. During
  sign up the child's name and age are saved to a profile row, and age is what
  drives the difficulty of the questions later on.
- **Pick a world.** From the home screen the child opens the biome picker
  (Forest, Savanna, Arctic), then picks one of the animals that live there, then
  picks a topic such as Math, Science, Space, Making Friends, or Emotional
  Learning.
- **Play.** The chosen animal asks AI generated questions about the chosen
  topic, and they adapt to the child's age and recent answers. Open ended
  answers are graded by the AI. Correct answers earn Stardust and build a
  streak. Three correct in a row makes the animal excited.
- **Spend Stardust.** The Space Shop sells cosmetics (hat, scarf, shoes) and
  gameplay helpers (Streak Savers and Small Hints). An AI robot shopkeeper chats
  with the child and answers free form questions through the same language
  model.
- **Earn rewards.** A spin the wheel reward can appear at the end of a session,
  sized by how the child did.

## AI question types and age customization

All of the AI work lives in `src/lib/ai.js`. Every question is generated by the
model for a specific age group, and the age group also decides which kinds of
questions the AI is allowed to write.

Age is bucketed into three groups:

- **3 to 5:** multiple choice only, with very simple words and short sentences.
- **5 to 8:** multiple choice, plus true or false and open ended text questions.
  Ages 6 to 9 also fall into this group.
- **10 and up:** the same three question types, but with richer vocabulary and
  questions that ask for real reasoning.

There are three question types:

- **Multiple choice.** Three answers, one correct. The answers are shuffled on
  the client so the correct one is not always in the same spot.
- **True or false.** A single statement that the child judges as true or false.
- **Text.** An open ended question the child answers in their own words.

Text answers cannot be graded on the device, so they are sent back to the AI for
evaluation. To keep things fast, the same request also asks the AI for the next
question. That means one round trip both grades the child's answer and returns
the next question, so the next question is ready to show and read aloud right
after the feedback. This is handled by `evaluateTextAndGenerateNext` in
`src/lib/ai.js`.

Because the AI sometimes wraps its reply in markdown or adds stray text, the
responses are run through a small parser that pulls out the JSON object before
it is used.

## Text to speech and sound

Questions and feedback are read out loud so children who cannot read yet can
still play.

- **Speech** (`src/lib/speech.js`) calls a backend speech endpoint that proxies
  Deepgram, so the API key stays on the server. Audio is cached by text, and the
  game prefetches the audio for a question before revealing it so the reveal and
  the voice line happen together. There is no awkward silent gap.
- **Sound effects** (`src/lib/sound.js`) cover correct and wrong chimes, a click
  sound on every click anywhere in the app, and looping ambient sounds that play
  while hovering biomes on the home screen.

## Tech stack

- **React 19** with **React Router** for the pages and navigation.
- **Vite** for the dev server and the production build.
- **Tailwind CSS v4** for styling, wired in through the Vite plugin.
- **shadcn style UI components** built on Radix primitives, kept in
  `src/components/ui`.
- **Supabase** for authentication and the database, accessed through a small set
  of CRUD helpers.
- **AI and speech backend.** A hosted service at
  `https://mh-backend-eight.vercel.app` exposes an `/api/ai` endpoint for text
  generation and an `/api/speech` endpoint for text to speech. The frontend only
  ever talks to these endpoints, so no model keys live in the client.

## Data model

User data is stored in a Supabase table called `profiles`, keyed by `user_id`.
The columns the app reads and writes are:

- `name` and `age` set at sign up. Age drives question difficulty.
- `stardust` the player's currency, always read from the database rather than
  hardcoded.
- `hat`, `scarf`, `shoes` owned cosmetics.
- `streak_saver`, `small_hint` owned gameplay helpers, stored as counts.

The database layer in `src/lib/database.js` exposes generic helpers
(`create`, `getById`, `list`, `update`, `upsert`, `remove`, `count`) so pages do
not write raw Supabase queries.

## Getting started

You need Node.js and npm.

1. Install dependencies:

   ```
   npm install
   ```

2. Create a `.env.local` file in the project root with your Supabase project
   values:

   ```
   VITE_SUPABASE_URL=your-project-url
   VITE_SUPABASE_PUBLISHABLE_KEY=your-publishable-key
   ```

3. Start the dev server:

   ```
   npm run dev
   ```

4. Build for production and preview the build:

   ```
   npm run build
   npm run preview
   ```

The AI and speech endpoints are hosted on a seperate backend, so no extra API keys are needed to run.

## Scripts

- `npm run dev` start the Vite dev server with hot reloading.
- `npm run build` create a production build in `dist`.
